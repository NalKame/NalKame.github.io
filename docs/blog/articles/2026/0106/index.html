Excelの列名と列数を変換するコードを書いてみました。 以下に列数を数値でいれるか、列名を大文字アルファベットで入れると変換してくれます。

列名から列数は正直に26×(何桁目か-1)×(何個目のアルファベットか)の和で計算できますが、 列数から列名に変換しようとすると、A=1,Z=26と0がないことから、 剰余で一桁目を求めるために1引いてから剰余を出して1足した値でアルファベットに変換する必要があります。 excel_column.jsの以下の箇所になります。 

number--; // 0始まりに調整
code = String.fromCharCode(65 + (number % 26)) + code;
number = Math.floor(number / 26);

この方式はbijective numerationというらしい。 （wikipedia参考）

0がないことにより、前ゼロを考慮しなくてよくなり、ありうる文字列と自然数が完全に1対1対応するものである。 10進を考えるとわかるが0を含まない数字は通常つかうものと同じであるが、0を含む場合は上の桁から借りてきて表記を変える必要がある。 さらに1進数というのも可能になっている。1を並べた数がその数であり非常に単純であるがおもしろい。 小数は桁を増やす時にかならず最小の値が1ではいってしまうため破綻しているという特徴もある。

bを基数、dnを桁ごとの値とすると、 Σndnbnで数字をあらわすというのが位取り記数法の基本的な形であり、 今回はdnのとりうる範囲を+1ずらしたことになる。 bやdの範囲の変更で様々な拡張がありうるようであり、大変おもしろい。 b=3でd={-1,0,1}とする平衡三進法は負数へも自然に拡張されるし、 bを黄金比にして小数の形式も使って整数を表したり、bを複素数にしてガウス整数を表現したり、いろいろ広がりがある。
